# Quickstart

This guide walks you through the fastest path to generating your first AI-powered video with MoneyPrinter.

## Prerequisites

Ensure you have the following installed:

- **Python 3.11+**
- **uv** package manager ([installation guide](https://docs.astral.sh/uv/getting-started/installation/))
- **FFmpeg**
- **ImageMagick**
- **Ollama**

<Note>
For detailed installation instructions, see the [Installation Guide](/setup/installation).
</Note>

## Setup Steps

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/FujiwaraChoki/MoneyPrinter.git
    cd MoneyPrinter
    ```
  </Step>

  <Step title="Run the interactive setup script (recommended)">
    The setup script checks dependencies, creates `.env`, installs packages, and optionally pulls an Ollama model:

    ```bash
    ./setup.sh
    ```

    **Or use manual setup:**

    ```bash
    # Install dependencies
    uv sync

    # Create environment file
    cp .env.example .env
    ```

    Windows PowerShell:
    ```powershell
    Copy-Item .env.example .env
    ```
  </Step>

  <Step title="Configure required environment variables">
    Edit `.env` and set:

    ```bash
    # Required
    TIKTOK_SESSION_ID="your_tiktok_session_id"
    PEXELS_API_KEY="your_pexels_api_key"

    # Optional (defaults shown)
    OLLAMA_BASE_URL="http://localhost:11434"
    OLLAMA_MODEL="llama3.1:8b"
    ```

    <Accordion title="How to get TikTok session ID">
      1. Log into TikTok in your browser
      2. Open Developer Tools (F12)
      3. Go to Application → Cookies
      4. Copy the value of the `sessionid` cookie
    </Accordion>

    <Accordion title="How to get Pexels API key">
      1. Create a free account at [pexels.com/api](https://www.pexels.com/api/)
      2. Generate an API key from your dashboard
    </Accordion>
  </Step>

  <Step title="Start Ollama and pull a model">
    In a new terminal:

    ```bash
    ollama serve
    ```

    Pull a model (if not already installed):

    ```bash
    ollama pull llama3.1:8b
    ```

    Verify the model is available:

    ```bash
    ollama list
    ```

    <Info>
    If Ollama runs on another machine or port, set `OLLAMA_BASE_URL` in `.env`.
    </Info>
  </Step>

  <Step title="Start the backend API">
    ```bash
    uv run python Backend/main.py
    ```

    The API will start on `http://localhost:8080`.
  </Step>

  <Step title="Start the worker (new terminal)">
    The worker claims jobs from the queue and runs the generation pipeline:

    ```bash
    uv run python Backend/worker.py
    ```
  </Step>

  <Step title="Start the frontend (new terminal)">
    ```bash
    cd Frontend
    python3 -m http.server 3000
    ```

    Open your browser to `http://localhost:3000`.
  </Step>

  <Step title="Generate your first video">
    In the web interface:

    1. Enter a video subject (e.g., "Top 3 AI business ideas")
    2. Expand **Advanced Options**
    3. Select an Ollama model from the dropdown
    4. Choose a voice (default: `en_us_001`)
    5. Click **Generate**

    Progress events will stream in real-time. The final video will be saved as `output.mp4` in the project root.
  </Step>
</Steps>

## Verify Installation

Test the API endpoints:

```bash
# List available Ollama models
curl http://localhost:8080/api/models

# Queue a test generation job
curl -X POST http://localhost:8080/api/generate \
  -H "Content-Type: application/json" \
  -d '{
    "videoSubject": "AI business ideas",
    "aiModel": "llama3.1:8b",
    "voice": "en_us_001",
    "paragraphNumber": 1,
    "customPrompt": ""
  }'
```

## Optional: Run Tests

Install dev dependencies and run the test suite:

```bash
uv sync --group dev
uv run pytest
```

See [Testing](/development/testing) for more details.

## Troubleshooting

<AccordionGroup>
  <Accordion title="No Ollama models in dropdown">
    - Ensure Ollama is running: `ollama serve`
    - Check models are installed: `ollama list`
    - Verify `OLLAMA_BASE_URL` in `.env`
  </Accordion>

  <Accordion title="ImageMagick not detected">
    Set the explicit path in `.env`:

    **Linux/macOS:**
    ```bash
    IMAGEMAGICK_BINARY="/usr/local/bin/magick"
    ```

    **Windows:**
    ```bash
    IMAGEMAGICK_BINARY="C:\\Program Files\\ImageMagick-7.1.1-Q16-HDRI\\magick.exe"
    ```

    Note the double backslashes (`\\`) for Windows paths.
  </Accordion>

  <Accordion title="Worker not processing jobs">
    - Ensure the backend API is running first
    - Check `DATABASE_URL` is correctly set (defaults to SQLite)
    - Look for errors in the worker terminal output
  </Accordion>
</AccordionGroup>
